allow cors - access-control-allow-origin welcome page

thanks for installing. allow cors is a flat, lightweight developer utility to bypass cors policies, strip security headers, and configure custom redirects.

important: pin the extension

to make toggling the extension and managing rules easy, please pin it to your browser toolbar:

  • click the puzzle piece icon (extensions menu) in the top-right of chrome.
  • find allow cors in the list.
  • click the pin icon next to it so it stays visible.
core features overview
  • smart credentials bypass: automatically matches the origin of your active tab (e.g., vercel or localhost) and injects access-control-allow-credentials: true to support secure requests.
  • auto-disable alarm: set a timer (e.g. 10m) to automatically turn off the extension. this prevents security issues if you forget to disable it while checking emails or doing personal banking.
  • redirect mapping: map remote API or script requests (like analytics trackers or production files) to local mocks (e.g., http://localhost:3000).
  • modes selection: switch between blocklist (bypass all except custom domains) and allowlist (only bypass specific domains).
  • security header stripping: toggle to strip Content Security Policy (CSP) and X-Frame-Options to allow framing/embedding.
